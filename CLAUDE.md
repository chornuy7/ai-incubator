# CLAUDE.md — гид по проекту для Claude Code

AI-платформа массового управления Telegram-профилями (прогрев, нейрокомментинг,
нейрочатинг, парсинг каналов, автоматизации, логи, аналитика). Фронт — React/Vite,
бэкенд — Node/Express (`server/`), сессии Telegram через GramJS.

## ⚠️ Читать первым делом (для задач по платформе/ТЗ)

1. **[docs/TZ-2026-07-10.md](./docs/TZ-2026-07-10.md)** — техническое задание (источник истины).
2. **[ROADMAP.md](./ROADMAP.md)** — единый роадмап: Часть I — план по ТЗ (фазы + 2 дорожки), Часть II — текущий статус.

**Обязанность Claude:** перед работой над задачей из ТЗ — свериться с этими двумя файлами;
после выполнения пункта — **обновить его статус в `ROADMAP.md`** (`☐→◐→☑`) в том же PR.
Оба документа коммитятся и пушатся вместе с кодом.

## Разделение работы (2 человека, 1 роадмап)

Работа идёт по двум дорожкам, разделённым по владению кодом (детали и файлы — в ROADMAP):

- **Lane A — Ядро:** профили, статусы, блокировки/lease, прогрев, задачи/оркестрация, прокси-модель.
- **Lane B — Кампании/UX:** цели/KB/CRM/нейрочатинг, нейрокомментинг, выбор каналов, парсер, логи, Help Center.

**Правило:** не редактировать файлы чужой дорожки в параллельных задачах. Общие точки
(`server/modules/workers.js`, схемы данных) — только через контракт из Фазы 0 ROADMAP.
Каждая задача — своя ветка `feat/<lane>-<имя>`, PR в `main`.

## Направление архитектуры (меняется — делать аккуратно)

- Явная **state machine** статусов аккаунта: `Active/Warming/Pause/FloodWait/Quarantine/SpamBlock/Reauth/Invalid`.
- **Блокировки → lease** («один канал — один бот», приоритет прогрева, аудит переноса).
- **Цели → База знаний → Лиды/CRM** как отдельный слой; AI всегда работает «к цели».
- **Адаптер платформы** (Telegram/Instagram) — Telegram-специфику не тащить в общий слой.
- Не ломать существующее: многое уже есть (см. ROADMAP → «Что уже реализовано»).

## Терминология (единая)

- **Профиль** — человек/пользователь системы. **Аккаунт** — управляемый бот. (Не путать.)
- **Логи**, не «История». **Шаблоны поиска**, не «Шаблоны».

## Команды и верификация

- Запуск дев-сервера: `npm run dev` (фронт :5173, API :3001). Дев-сервер — только через harness preview, не через фоновый bash.
- Типы: `npx tsc --noEmit`. Тесты: `npm test`.
- **После правок воркеров/схем бэкенд надо перезапускать** — воркеры живут в памяти процесса; при рестарте running-задачи помечаются `stopped` (`reconcileStaleTasksOnBoot`).

## Память и навигация по коду

- **codebase-memory-mcp** — граф кода (tree-sitter) для быстрой навигации по большой кодовой базе.
  Ставит каждый разработчик у себя (автоконфигурит Claude Code через `.mcp.json`). Команды установки:
  ```bash
  # Linux/macOS
  curl -fsSL https://raw.githubusercontent.com/DeusData/codebase-memory-mcp/main/install.sh | bash
  # + опциональный 3D-UI графа на localhost:9749
  curl -fsSL https://raw.githubusercontent.com/DeusData/codebase-memory-mcp/main/install.sh | bash -s -- --ui
  ```
  ```powershell
  # Windows (PowerShell)
  Invoke-WebRequest -Uri https://raw.githubusercontent.com/DeusData/codebase-memory-mcp/main/install.ps1 -OutFile install.ps1
  Unblock-File .\install.ps1; .\install.ps1
  ```
  > Это удалённый install-скрипт — запускает его **человек** осознанно, Claude его не выполняет.
  > После установки перезапустить Claude Code и сказать «Index this project».
- **Файловая память Claude** (`.claude/.../memory/`) — фиксировать решения, договорённости и контракты между дорожками.
